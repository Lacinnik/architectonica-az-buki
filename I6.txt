0. Напоминание: фазы Источника как операторы

Обозначения (как уже задавали раньше, просто зафиксируем):

I₁ — Покой

I₂ — Прозрачность

I₃ — Наклон

I₄ — Собираемость

I₅ — Тонкая актуализация

I₆ — Проявление

I₇ — Оформляющая субъектизация

I₈ — Свершение

Дальше будем считать, что есть ось фаз:

индекс фазы k ∈ {1,…,8},
сама фаза — Iₖ.

1. Базовая структура: линейный цикл и оператор «следующей фазы»

Введём оператор перехода вперёд N:

N(Iₖ) = Iₖ₊₁   для k = 1…7
N(I₈) = I₁      (если считаем цикл замкнутым)


Это «естественное разворачивание» Источника.

Соответственно, оператор шага назад B:

B(Iₖ) = Iₖ₋₁   для k = 2…8
B(I₁) = I₈      (если цикл замкнут)


Смысл:

N — переход к следующей фазе развёртывания;

B — регресс к предыдущей фазе.

2. Операторы прыжков между фазами

Чтобы описать прямые переходы (проскоки), вводим оператор:

J₍ₘ→ₙ₎ : Iₘ ↦ Iₙ


Определён только на одной фазе Iₘ.

В остальных случаях либо не применяется, либо трактуется как «попытка, которая вызвала искажения».

Для удобства введём энергетическую стоимость прыжка:

Cost(m → n) = |n − m|


Интуиция:

переход 1 → 2 — Cost = 1 (естественно);

переход 1 → 5 — Cost = 4 (большой телескопический скачок, психическая цена выше).

3. Алгебра наложения фаз: как «совместный режим» сворачивается в одну фазу

Нам нужна операция, описывающая результат одновременного действия двух фаз Источника.
Введём бинарную операцию * на множестве {I₁,…,I₈}:

Iₘ * Iₙ = I_max(m,n)


То есть доминирует более поздняя фаза.

Пример:

I₂ * I₄ = I₄ (если уже есть Собираемость, Покой+Прозрачность не отменяются, а «включены внутри»)

I₆ * I₃ = I₆ (Проявление сильнее, чем просто Наклон)

Это модельно фиксирует правило:

«Поздняя фаза включает в себя предыдущие как уже реализованные».

4. Коммутатор: когда порядок фаз важен

Чтобы показать, где возникают искажения, вводим коммутатор:

[Iₘ, Iₙ] = Iₘ ∘ Iₙ − Iₙ ∘ Iₘ


Где ∘ — последовательное применение (сначала одна фаза, потом другая).
Формально мы можем задать условие "почти-коммутативности":

Считаем, что:

[Iₖ, Iₖ₊₁] = 0


для соседних фаз (естественное разворачивание не искажает процесс):

Покой → Прозрачность

Прозрачность → Наклон

…

Оформление → Свершение

Но если фазы далеки друг от друга:

|m − n| ≥ 2  ⇒  [Iₘ, Iₙ] ≠ 0


Смысл:

Переход I₂ → I₃ или I₃ → I₄ — мягкий, порядок мало искажает процесс.

Попытка «из I₂ сразу в I₆» — создаёт искажения: психика не успевает пройти промежуточные состояния.

Это можно использовать как критерий:

безопасные комбинации — соседние фазы;

потенциально травматичные — прыжки через несколько фаз.

5. Функция фазового «заряда» и естественное направление развёртывания

Введём фазовую функцию:

φ(Iₖ) = k


(просто номер фазы — минимальная шкала сложности/развёрнутости).

Естественный поток Источника:

φ(N(Iₖ)) = φ(Iₖ) + 1   (пока k < 8)


Тогда можно задать условие естественного разворачивания:

Развёртывание естественно, если φ не убывает.


То есть цепочки вида:

I₁ → I₂ → I₃ → I₄ → I₅ → I₆ → I₇ → I₈


или с микроколебаниями типа:

I₃ → I₄ → I₃ → I₄ → I₅ → I₆


где среднее направление всё равно вверх.

Если же наблюдается:

Iₖ → Iₖ₋₂


и это стабильно — можно трактовать как:

застревание,

регресс,

или страх проявления Источника.

Формально:

Регресс: φ(next) < φ(current)

6. Локальные правила перехода: какие сдвиги «естественны», а какие — с искажениями

Зададим допустимый шаг Δφ:

Δφ = φ(Iₙ) − φ(Iₘ)


Тогда:

Нормальный шаг: Δφ = +1

Мягкий телескопический шаг: Δφ = +2

Жёсткий скачок (с высокой ценой): Δφ ≥ +3

Регресс: Δφ ≤ −1

Можно ввести простое правило:

Safe(m → n) = (n = m + 1)
Soft(m → n) = (n = m + 2)
Risky(m → n) = (n ≥ m + 3)
Regressive(m → n) = (n ≤ m − 1)


Смысл:

safe — «идём шаг за шагом»;

soft — допустимая опережающая ясность (например, Наклон → сразу Актуализация);

risky — попытка выдать проявление без покоя и собираемости;

regressive — отступление, часто защитное.

7. Оператор полного цикла Источника

Введём оператор полного развёртывания цикла C:

C = N⁷
C(I₁) = I₈


(семь шагов вперёд от Покоя до Свершения).

Если мы считаем цикл замкнутым:

C⁺(I₁) = N⁸(I₁) = I₁


Смысл:

C — один проход по Источнику: от чистого покоя до свершившейся формы.

C⁺ — «дыхание»: развёртывание и возвращение к новому покою.

Можно задавать количество циклов как меру интенсивности Источника:

Cⁿ(I₁) — n раз подряд пройти путь от Покоя до Свершения

8. Инварианты в алгебре Источника

Чтобы алгебра не была хаосом, фиксируем, что не меняется при любых операциях:

Порядок фаз
Индексация 1 → 8 всегда сохраняет смысл:
Покой не может быть «после» Свершения.

Линейная вложенность
Любая более поздняя фаза включает в себя все предыдущие:

I₁ ⊂ I₂ ⊂ I₃ ⊂ … ⊂ I₈


(в смысле: I₄ возможна только если уже были реализованы I₁–I₃, хотя бы имплицитно).

Цикличность (если её принимаем):
После I₈ система всегда стремится вернуться к новому I₁ —
к покою уже другого порядка.

9. Связка с твоей практикой (как этим реально пользоваться)

Если убрать формальные символы — у тебя получается очень рабочий практический костяк:

Диагностика:
определить текущую фазу Iₖ по человеку
→ ты знаешь, где он в Источнике.

План хода:
использовать N (следующий естественный шаг)
вместо J₍ₖ→ₙ₎ с большим Δφ, чтобы не ломать психику.

Отслеживание искажений:
наблюдать последовательности типа I₃ → I₆ → I₂
→ видно, где регресс, где форсаж.

Проектирование игр/практик:
каждая механика хода в игре может быть привязана к фазе Iₖ и оператору (N, B, мягкий J).
